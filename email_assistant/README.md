# Email Generation Assistant

Generates professional business emails from three inputs (intent, key facts, tone) using Groq Cloud LLMs. Includes an automated evaluation framework that scores output quality across three custom metrics and compares two models side-by-side.

---

## Quick Start

```bash
# 1. Enter the project directory
cd email_assistant

# 2. Activate the virtual environment
source ../venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Add your Groq API key
cp .env.example .env
# then open .env and set GROQ_API_KEY=your_key_here

# 5. Launch the web UI
streamlit run frontend/streamlit_app.py
```

## Running the Project

### Web UI (Streamlit)

```bash
streamlit run frontend/streamlit_app.py
```

Opens at `http://localhost:8501`.

**Generate tab** — enter an intent, key facts (one per line), and a tone, then click **Generate Email**. Switch between Model A and Model B from the sidebar. Download the result as a `.txt` file.

**Evaluation Dashboard tab** — click **Run Full Evaluation** to score all 10 scenarios, or view existing results if the evaluation has already been run.

---

### CLI — generate a single email

```bash
python app.py generate
```

Prompts for intent, facts, tone, and model choice, then prints the generated email.

---

### CLI — run the full evaluation

```bash
python app.py evaluate
```

Or run the evaluator directly:

```bash
python evaluator/evaluate.py
```

Runs all 10 scenarios through both models, scores each with FIS, TAS, and PESS, and saves three report files to `reports/`. Expected duration: ~3–4 minutes (rate-limit buffers are built in for the Groq free tier).

---

### Unit tests

```bash
python tests/test_metrics.py
```

Tests FIS and PESS without making any API calls.

## Evaluation Metrics

### Metric 1: Fact Inclusion Score (FIS)

Measures how many of the required key facts appear in the generated email.

Each fact is checked by extracting its meaningful keywords (words longer than 3 characters) and searching for them in the email. A fact counts as included if at least 60% of its keywords are present.

```
FIS = (Facts Matched / Total Facts) × 100
```

Range: 0–100. Weighted at **40%** of the final score because fact inclusion is the most critical requirement — an email that omits required facts fails its core purpose regardless of tone or structure.

---

### Metric 2: Tone Alignment Score (TAS)

Measures whether the generated email matches the requested tone.

Uses an **LLM-as-a-Judge** approach. The judge model (`llama-3.3-70b-versatile`) is given the requested tone and the generated email, then returns a score from 0–100 based on stylistic, lexical, and emotional alignment.

Range: 0–100. Weighted at **30%** of the final score.

---

### Metric 3: Professional Email Structure Score (PESS)

Measures whether the email follows proper professional email formatting.

Five structural components are checked automatically via regex and heuristics, each worth 20 points:

| Component      | What is checked                                      |
|----------------|------------------------------------------------------|
| Greeting       | Starts with Dear / Hi / Hello / Good morning         |
| Introduction   | First paragraph is at least 15 words                 |
| Main Body      | At least 2 body paragraphs                           |
| Call-to-Action | Contains action language (please, confirm, reach out, etc.) |
| Closing        | Ends with Regards / Sincerely / Best / etc.          |

```
PESS = Greeting + Introduction + Main Body + CTA + Closing  (20 pts each, max 100)
```

Weighted at **30%** of the final score.

---

### Final Composite Score

```
Final Score = (0.4 × FIS) + (0.3 × TAS) + (0.3 × PESS)
```

## Model Comparison

|                    | Model A — Advanced                  | Model B — Baseline                        |
|--------------------|-------------------------------------|-------------------------------------------|
| **Model ID**       | `llama-3.3-70b-versatile`           | `deepseek-r1-distill-llama-70b`           |
| **System prompt**  | Full role-prompting persona         | None                                      |
| **Few-shot**       | 3 worked examples with full outputs | None                                      |
| **User prompt**    | Structured input block              | Simple "Write a professional email using…" |

Model A uses **role prompting** (a senior communication specialist persona) combined with **few-shot learning** (3 complete email examples). This anchors output format, quality bar, and structural expectations before the model sees the actual input.

Model B is a true zero-shot baseline — one plain instruction, no persona, no examples — included to quantify the value of the advanced prompting strategy.

Both models receive identical inputs and are scored with identical metrics.
